/
├── etc/
│   └── mise/
│       └── config.toml # System-wide
└── home/
    └── <user>/
        ├── .config/
        │   └── mise/
        │       └── config.toml # User-specific (global)
        └── <workspace>/
            ├── mise.local.toml # Project-specific (local override)
            ├── mise.toml # Project-specific
            └── <service>/
                └── mise.toml # Service-specific

### Trusted config paths

:::info[References]

- [mise Docs / Configuration / Settings / trusted_config_paths](https://mise.jdx.dev/configuration/settings.html#trusted_config_paths)
- [mise Docs / Getting Started / Trusting config files](https://mise.jdx.dev/getting-started.html#trusting-config-files)

:::

```toml title="$HOME/.config/mise/config.toml"
[settings]
trusted_config_paths = ["/home/hhk7734"]
```

- `trusted_config_paths` defines configuration roots that `mise` should trust.
- With `trusted_config_paths = ["/home/hhk7734"]`, `mise.toml` files under `/home/hhk7734` and its subdirectories are trusted automatically.
- This trusts descendants of the configured path, not only the exact directory itself.

## 버전 관리

### 버전 설정

```shell
mise ls-remote <tool>
```

```shell
mise use [<flag>] [<backend>:]<tool>[@<version>]
```

- `<flag>`
  - `--pin`: 사용한 정확한 버전을 설정 파일에 기록합니다.
  - `-g|--global`: 전역 설정 파일에 기록합니다.
- `<backend>`
  - [mise Docs / Backends](https://mise.jdx.dev/dev-tools/backends/)

```shell
mise ls [<tool>]
```

### 버전 해제

```shell
mise unuse <tool>
```
