TDD를 통해 진행하는 과정에 변하면 안되는 시스템 특성을 유지

도메인 모델에 대한 테스트가 너무 많으면 변경이 어려움

도메인 모델보다 서비스 레이어에 중점을 두고 테스트하면 변경에 대한 유연함을 가져올 수 있음

서비스레이어에서 도메인 객체를 직접 인자로 받거나 리턴하지 않도록 하면 서비스 레이어 테스트의 도메인 모델에 대한 의존성을 줄일 수 있음
그렇게 하기 어려운 경우 테스트에서 도메인 객체 부분은 도우미함수나 픽스처에 넣어서 추상화
