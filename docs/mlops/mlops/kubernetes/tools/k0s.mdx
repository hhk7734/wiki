## 설치

- https://docs.k0sproject.io/latest/install/

```shell
curl -sSLf https://get.k0s.sh | sudo K0S_VERSION=v1.27.4+k0s.0 sh
```

## Controller

### start

```shell
k0s config create > k0s.yaml
```

- https://docs.k0sproject.io/latest/configuration/

```yaml title="k0s.yaml"
apiVersion: k0s.k0sproject.io/v1beta1
kind: ClusterConfig
metadata:
  name: k0s
spec:
  api:
    address: <host|ip>
    externalAddress: <host|ip> # 외부 접근용
    port: 6443
    k0sApiPort: 9443
    sans: # 인증서 SAN
      - <address>
      - <externalAddress>
      - <host|ip>
    tunneledNetworkingMode: false
  storage:
    type: etcd
    etcd:
      peerAddress: <address>
  network:
    clusterDomain: cluster.local
    podCIDR: 10.244.0.0/16
    serviceCIDR: 10.96.0.0/12
    provider: kuberouter
    calico: null
    kuberouter:
      autoMTU: true
      hairpin: Enabled
      ipMasq: false
      metricsPort: 8080
      mtu: 0
      peerRouterASNs: ""
      peerRouterIPs: ""
    kubeProxy:
      iptables:
        minSyncPeriod: 0s
        syncPeriod: 0s
      ipvs:
        minSyncPeriod: 0s
        syncPeriod: 0s
        tcpFinTimeout: 0s
        tcpTimeout: 0s
        udpTimeout: 0s
      metricsBindAddress: 0.0.0.0:10249
      mode: iptables
    nodeLocalLoadBalancing:
      envoyProxy:
        apiServerBindPort: 7443
        image:
          image: quay.io/k0sproject/envoy-distroless
          version: v1.24.1
        konnectivityServerBindPort: 7132
      type: EnvoyProxy
  podSecurityPolicy:
    defaultPolicy: 00-k0s-privileged
  controllerManager: {}
  telemetry:
    enabled: false
  installConfig:
    users:
      etcdUser: etcd
      kineUser: kube-apiserver
      konnectivityUser: konnectivity-server
      kubeAPIserverUser: kube-apiserver
      kubeSchedulerUser: kube-scheduler
  konnectivity:
    adminPort: 8133
    agentPort: 8132
  scheduler: {}
  extensions:
    helm:
      charts: null
      concurrencyLevel: 5
      repositories: null
    storage:
      type: openebs_local_storage
```

```shell
sudo k0s install controller -c k0s.yaml
```

- `--single`: controller + worker
- `--kubelet-extra-args <extraArgs>`: kubelet 옵션

<br />

- https://docs.k0sproject.io/latest/networking/

필요한 포트가 열려있는지 확인합니다.

| Protocol | Port  | Service               | Direction                 | Notes |
| -------- | ----- | --------------------- | ------------------------- | ----- |
| TCP      | 2380  | etcd peers            | controller<-\>controller  |       |
| TCP      | 6443  | Kubernetes API Server | worker=\>controller       |       |
| TCP      | 179   | kube-router           | worker<-\>worker          | BGP   |
| TCP      | 4789  | Calico                | worker<-\>worker          | VXLAN |
| TCP      | 10250 | kubelet               | master,worker=\>host "\*" |       |
| TCP      | 9443  | k0s-api               | controller<-\>controller  |       |
| TCP      | 8132  | konnectivity          | worker<-\>controller      |       |

:::info
램이 부족한 환경인 경우 swap을 사용할 수 있습니다.

- https://docs.k0sproject.io/latest/system-requirements/

```shell
sudo fallocate -l 3G /swapfile
```

```shell
sudo chmod 600 /swapfile \
&& sudo mkswap /swapfile \
&& sudo swapon /swapfile
```

:::

```shell
sudo k0s start
```

```shell
sudo k0s status
```

:::info

`sudo k0s status` 값이 안나오거나 에러가 있다고 나오는 경우 아래 명령어를 통해 실행 상태와 로그를 확인할 수 있습니다.

```shell
sudo systemctl status k0scontroller
```

```shell
journalctl -xe -u k0scontroller
```

:::

### kubeconfig

- https://docs.k0sproject.io/latest/user-management/

```shell
sudo k0s kubeconfig create <user>
```

- `--groups <group>`
  - `system:masters` 그룹은 기본적으로 cluster-admin ClusterRole에 바인딩되어 있습니다.

### stop

```shell
sudo k0s stop
```

다시 시작할 일이 없어 완전히 삭제하고 싶은 경우 아래 명령어를 사용합니다.

```shell
sudo k0s reset
```
