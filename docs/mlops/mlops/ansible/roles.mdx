## Role

### Directory structure

```shell
roles/
├── <category>
│   └── <role>
├── <role>
│   ├── defaults/
│   │   └── main.yaml
│   ├── files/
│   ├── handlers/
│   │   └── main.yaml
│   ├── meta
│   │   ├── argument_specs.yaml
│   │   └── main.yaml
│   ├── tasks/
│   │   └── main.yaml
│   ├── templates/
│   │   ├── <template>.j2
│   │   └── ...
│   ├── vars/
│   │   └── main.yaml
│   └── ...
└── ...
```

- `defaults/main.yaml`
  - 우선순위가 상대적으로 낮은 설정 값입니다
  - [변수 관리](/docs/mlops/mlops/ansible/variables.mdx)
- `vars/main.yaml`
  - 우선순위가 상대적으로 높은 설정 값입니다.
  - [변수 관리](/docs/mlops/mlops/ansible/variables.mdx)

## role 사용하기

### role 경로 설정

- `roles/` 디렉토리를 확인합니다.
- `ansible.cfg`의 `default.roles_path: <path>[:<path>]` 경로를 확인합니다.
  - `~/.ansible/roles:/usr/share/ansible/roles:/etc/ansible/roles`이 기본 경로입니다.

### role 호출

```yaml
- roles:
    - role: <role> # or <category>/<role>
      vars:
        <var>: <value>
      tags:
        - <tag>
      when: <condition>
```

```yaml
- tasks:
    - ansible.builtin.include_role:
        name: <role> # or <category>/<role>
      vars:
        <var>: <value>
        <var>: {{ loop_item }}
      loop:
        - <item1>
        - <item2>
      loop_control:
        loop_var: loop_item
```

```yaml
- tasks:
    - ansible.builtin.import_role:
        name: <role> # or <category>/<role>
```
